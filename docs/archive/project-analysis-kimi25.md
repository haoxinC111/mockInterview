## 一、项目定位回答 (基于7个核心问题)

### 1. 你到底在做什么？

InterviewSim 是一个**AI驱动的模拟面试训练系统**，核心功能是：

- **简历智能解析**: 提取PDF简历中的技能、项目经历，构建候选人画像
- **大纲驱动面试**: 根据目标岗位(Agent Engineer)生成结构化面试大纲(LLM基础→RAG架构→Agent编排)
- **动态追问引擎**: 基于用户回答质量(1-5分)决定深挖/切题/结束，支持多轮深度对话
- **多维度评估**: 技术深度、架构设计、工程实践、沟通表达四个维度评分
- **结构化报告**: 生成包含优势、风险点、薪资匹配度、30天行动计划的复盘报告

**技术亮点**:
- 3级回退策略: LLM → 启发式规则 → 硬编码默认
- 本地Whisper语音输入支持
- Feature flag控制LLM调用，零成本运行模式
- 完整的决策链路追踪(decision_traces)

### 2. 解决的是什么市场问题？

**当前解决的问题** (面向开发者自我训练):

| 痛点 | InterviewSim的解决方案 |
|------|------------------------|
| 找不到合适的面试练习对象 | 7x24可用的AI面试官 |
| 不知道自己回答质量如何 | 实时评分+证据+改进建议 |
| 不知道薄弱点在哪里 | 4维度雷达图+具体gaps |
| 面试后无法复盘 | 完整的对话trace+决策链路 |
| 缺乏针对性训练 | 基于简历定制大纲 |

**市场空白**:
- 现有产品(牛客网、LeetCode)侧重算法，缺少面向Agent Engineer的系统性面试训练
- 通用AI对话(ChatGPT)缺乏结构化评估和报告能力
- 真人Mock Interview成本高、难以规模化

### 3. 你的目标用户是谁？

**当前阶段 (v0.5) - 单用户自用**:
- 你自己: 准备Agent Engineer岗位面试的开发者

**潜在扩展用户群** (如需商业化):
- **初级-中级后端开发**: 转型Agent Engineer的工程师
- **应届/实习求职者**: 缺少面试经验，需要低成本练习
- **技术管理者**: 用于团队技术能力评估
- **培训机构**: 作为教学辅助工具

### 4. 产品在市场中的定位

**当前定位**: 个人训练工具 (Personal Training Tool)

**如果面向市场，可能的定位**:
```
                    专业度↑
                      │
         真人Mock     │    InterviewSim ← 可能位置
         Interview    │    (系统化+可复盘)
                      │
──────────────┼────────────────────────→ 可规模化
                      │
         LeetCode     │    ChatGPT
         算法练习     │    (通用对话)
                      │
                    专业度↓
```

**差异化优势**:
- 垂直场景深度: 专门针对Agent Engineer岗位
- 可解释性: 每个决策都有reason trace，不是黑盒
- 成长性导向: "帮助候选人成长"是核心使命，不是考倒用户

### 5. 商业模式是否清晰？

**当前**: 无商业模式，纯自用工具

**潜在商业模式分析**:

| 模式 | 可行性 | 挑战 |
|------|--------|------|
| SaaS订阅 | 中等 | LLM调用成本高，需要平衡成本与定价 |
| 企业服务(招聘 screening) | 中等 | 需要企业级功能(ATS集成、多用户管理) |
| 教育培训合作 | 较高 | 与编程培训机构合作，按学员收费 |
| 开源+服务 | 较低 | 代码量较大，社区维护成本 |

**成本结构考量**:
- LLM调用成本: 每次面试约10-20轮对话，约$0.1-0.3/次
- 本地运行优势: Whisper本地推理、SQLite存储，边际成本接近0

### 7. 单位经济是否成立？

**单位经济分析 (估算)**:

假设每次完整面试(12轮对话):
```
收入侧:
  - 假设订阅制: ¥50/月，用户平均使用5次
  - 单次收入: ¥10

成本侧:
  - LLM调用: 12轮 × 2K tokens × ¥0.003/1K = ¥0.07
  - 服务器成本: 分摊可忽略
  - 边际成本: ~¥0.1/次

毛利: ~99%
```

**结论**: 单位经济成立，LLM成本占比很低。关键是获客成本(CAC)和留存率。

## 二、代码质量评价

### 优势 ✅

1. **架构清晰，分层明确**
   - API层(routes.py) + 服务层(services/) + 模型层(models/) 分离良好
   - InterviewEngine核心逻辑封装完整，便于单元测试

2. **Feature Flag设计优秀**
   - `RESUME_PARSER_USE_LLM` / `INTERVIEW_ENGINE_USE_LLM` / `INTERVIEW_TURN_USE_LLM`
   - 支持零成本运行(纯规则模式)到全LLM模式的平滑切换

3. **可观测性良好**
   - 完整的日志系统(log_event + log_summary)
   - 决策链路追踪(decision_traces)
   - workflow API可导出完整面试过程

4. **容错机制完善**
   - LLM调用失败自动回退到规则模式
   - 3级大纲构建: LLM → 简历信号启发式 → 默认模板
   - resume解析缓存机制

5. **数据模型设计合理**
   - SQLModel兼顾类型安全和ORM便利
   - JSON列存储复杂嵌套数据(state_json, profile_json)
   - ResumeParseCache避免重复解析

### 改进空间 ⚠️

1. **测试覆盖不足**
   - 仅2个测试文件，4个测试用例
   - 缺少InterviewEngine核心逻辑的单元测试
   - 缺少LLM fallback路径的测试

2. **代码重复**
   - `llm_client.py`中`chat_json`和`chat_json_sync`大量重复代码
   - routes.py中多个endpoint有类似的db查询逻辑

3. **错误处理可以更细化**
   - 部分Exception捕获过于宽泛
   - 缺少对特定错误类型的处理(如LLM rate limit)

4. **配置管理可以加强**
   - 部分硬编码配置(DEFAULT_OUTLINE, KNOWN_SKILLS)可以考虑外置
   - 缺少配置验证

5. **前端代码组织**
   - app.js接近1000行，功能耦合
   - 缺少模块化拆分

## 三、核心优化建议

### 高优先级 🔴

#### 1. 增加测试覆盖
```python
# 建议增加以下测试:
- test_interview_engine_turns.py  # 测试各种decision分支
- test_llm_client_fallback.py      # 测试模型降级逻辑
- test_resume_parser.py            # 测试各种简历格式
- test_report_service.py           # 测试报告生成
```

#### 2. 代码重构
- 提取`llm_client.py`中的公共HTTP逻辑
- 将routes.py按功能拆分为多个router文件
- 前端app.js按功能模块拆分

#### 3. 配置外置化
- 将`DEFAULT_OUTLINE`、`KNOWN_SKILLS`移入配置文件或数据库
- 支持运行时更新面试大纲模板

### 中优先级 🟡

#### 4. 增强评估维度
当前4维度评估在`TurnEvaluation`中有定义，但`report_service.py`中实现较简单:
```python
# 当前代码只使用了简单的平均分:
"dimension_scores": {
    "technical_depth": round(avg, 2),
    "system_design": round(max(avg - 0.3, 0), 2),
    "communication": round(min(avg + 0.2, 5), 2),
}

# 建议: 让每个topic关联主维度和辅助维度
# 在evaluate时根据关键词命中情况做维度映射
```

#### 5. 面试历史分析
当前已有session列表和workflow trace，可以:
- 生成用户能力成长曲线
- 识别反复出现的问题模式
- 推荐针对性练习

#### 6. 题库/知识库
当前大纲是固定的，可以:
- 建立可扩展的题库系统
- 支持自定义面试题
- 题目难度分级

### 低优先级/探索性 🟢

#### 7. 语音输出(TTS)
当前只有语音输入(STT)，增加TTS:
- 让面试官"开口说话"
- 更真实的面试体验
- 技术选型: edge-tts(免费) / 讯飞(中文好)

#### 8. 多模态支持
- 简历支持图片格式(OCR)
- 支持白板/画图题(系统设计)

#### 9. 社区功能
- 面试题分享
- 优秀回答案例库
- 用户间互评
